Cloth and Ball Interaction

Link to Code

    For my first cloth simulation, I built an physical interaction simulation between a cloth and a sphere. The sphere is controlled with the mouse and the cloth realistically reacts to when it comes in contact with the ball. One problem that arose while I was working on this simulation is that if the ball came into contact with the net too fast, it would just go right through. This is because my collision check function was only being called once per draw loop. To fix this issue, I created two seperate loops in my draw function, one for displaying the cloth and the ball, and another loop for checking checking collisions. The collision check function now is being called 60 times per draw loop instead of the one time, which allows for more 60 times more collision detection checks per second then before, allowing for a more realistic simulation. For the 3D user controlled camera I am using the Peasycam library for processing.

   The video to the right shows the simulation in action. It shows the 3d navigatable, user-controlled camera of the scene. The first figure in the upper left is the framerate, and the second figure is the number of stiches in the cloth (rows * columns). When the user presses 'd' the cloth becomes unpinned to the corners and gravity takes control. When the cloth comes into contact with the ball, a drag force is applied and the cloth slowly falls off the ball and onto the ground.

   The bottom video on the right shows how I have implemented tearable cloth. When the ball comes into contact with the cloth, the links between each stich that comes into contact with the ball becomes unconnected, resulting in a tearable cloth simulation.

Flag in Wind Simulation

Link to Code

   To simulate a wind component of drag term, I decided to create a flag simulation. To achieve a realistic wind force for the flag, I used perlin noise along a 3D space, with three different offset counters. An x offset that would reset to 0 every draw loop, a y offset that would rest every iteration of the stich loop within the draw loop, and lastly a z offset counter that never gets reset. These in combination with the noise() function of processing, provided for a realisitc wind force being applied. I have included a flag pool and grass for context in the simulation.
